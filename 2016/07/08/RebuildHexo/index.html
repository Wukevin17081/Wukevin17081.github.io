RebuildHexo

發表於 2016-07-08

因為我之前的電腦出了問題，所以我硬碟的資料有很多都不見了，Blog 也是其中之一。 所以我必須整個重建整個環境，因為這個過程其實是太痛苦了，所以把他記錄下來，希望下一次不會這麼麻煩，或者不要用到更好。

首先，因為資料都不見了，所以第一個想到的是從 github 上 pull 資料， 但是上面的只有已經改為 html 的資料以及網頁的 css 等設定檔，並沒有其他東西。所以我還是必須將整個重裝， 重設定， 最多就是不必重打字，但還是要重新做 markdown 的檔案。

ssh key

先按照之前寫的建立 用 hexo 建一個 blog 把環境建起來。這其中有一些東西我沒有寫好。
在建立 ssh key 的地方，之前是寫說如果沒有的話建立一個新的，但是我沒有說如果有的話怎麼辦。
其實我並沒有用一個已存在的 key 來用，都是用一個新的。但是在產生 key 的時候，因為有地方弄錯，所以我必須重來一次。 所以我查了一下怎麼刪除 ssh key.
在 git bash 裡面輸入

hostname 指的是在 known_hosts 這個檔案裡面開頭的第一段。 一般來說都是 github.com , 刪除之後再重建一次key, 然後把 ssh key 輸入到 github 的網頁裡。

Hexo 部署

在之前， repository 我是寫 git@github.com:github帳號/github帳號.github.com.git 但是這是錯的， 正確的應該是 git@github.com:github帳號/github帳號.github.io 這個地方我試了很久，最後才試出來，可能是 github 改了什麼，因為我在網路上看到大家都是寫 .com.git, 但是我的就是不會過。最後我回去看我 github 的帳號才覺得不對，就改看看，然後就過了。

blog 主題

這一部份出乎意料的久，我是比較偏好深色底的，但是不知道為什麼，主流好像都是白色，之前有找了一個黑色調的，但是用起來有一些問題，所以這次就想說換一個好了，但找到不是佈局不喜歡，就是字體的顏色是用莫名淺的灰，白色應看久了都已經很吃力了，再用這麼淺的顏色，更難過。到最後我用的是 NexT 這套，其實這個也沒有什麼大問題，只是它有點窄，應該可以再放寬一點，文章移到正中間，現在的有點偏右，這樣就很好，就是是黑色調就完美了，只可惜我不知道怎麼改，不知道這會不會很難，改顏跟放寬一點應該還好，有空來試看看。

備份

其實這一部份我還沒做，但是我應該要做，因為我完全了解了之前有些網站會強調他們有定期備份這個功能。 因為只要丟掉一次就瘋了。幸好我的文章不算多，要是像別人有幾百篇，還是幾千篇，那種的不見了怎麼辦。 因為這次硬碟壞掉，所以我還特地去買了一個 nas，本來想說我也沒有什麼重要的資料，不見就不見了，但是，就算沒有重要的資料，有些東西不見了還是很麻煩。還是買一個來備份好了，雖然說荷包也大失血了。
